## 📋 Module Overview

The RBAC (Role-Based Access Control) & Security module is the backbone of eh2r AI's permission system. It allows Super Admins to define roles, configure granular permissions for each module, and maintain comprehensive audit logs. The module supports hierarchical role levels, module-level CRUD permissions (read, write, modify, delete, admin), and system roles that cannot be deleted. Audit logs track all user actions for compliance and security monitoring.

The RBAC & Security module is accessible from both the sidebar navigation and the module home page grid. It contains the following sub-items that provide specialized functionality within the module.

---

## 🔧 Sub-Items & Features

### 1. Role Master

Create and manage roles with configurable fields including name, description, hierarchy level, default dashboard, visible menu items, UI color, and granular permissions. System roles (isSystem=true) cannot be deleted.

**Steps to use Role Master:**
1. Navigate to the RBAC & Security module from the sidebar or module home page
2. Click on "Role Master" from the sub-menu
3. Use the available options to create, view, edit, or manage records
4. Apply filters or search as needed to find specific items
5. Save or submit your changes when complete

### 2. Permissions

Define module-level permissions for each role with CRUD access control (read, write, modify, delete, admin). Permissions can be configured per functional area (HR, Payroll, Attendance, Performance, Learning, Analytics).

**Steps to use Permissions:**
1. Navigate to the RBAC & Security module from the sidebar or module home page
2. Click on "Permissions" from the sub-menu
3. Use the available options to create, view, edit, or manage records
4. Apply filters or search as needed to find specific items
5. Save or submit your changes when complete

### 3. Audit Logs

Comprehensive log of all user actions including login events, data modifications, permission changes, and administrative actions. Each log entry includes timestamp, user, action type, and affected resource.

**Steps to use Audit Logs:**
1. Navigate to the RBAC & Security module from the sidebar or module home page
2. Click on "Audit Logs" from the sub-menu
3. Use the available options to create, view, edit, or manage records
4. Apply filters or search as needed to find specific items
5. Save or submit your changes when complete

## 💡 Tips & Best Practices

1. Follow the principle of least privilege — grant only the minimum permissions required for each role.
2. Regularly audit role permissions to ensure they align with current organizational needs.
3. Use system roles as templates when creating custom roles to maintain consistency.
4. Review audit logs periodically to detect any unauthorized access attempts.
5. Document role configurations for compliance and onboarding purposes.
